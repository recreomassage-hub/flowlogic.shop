# Команда: /qa

Запускает QA проверку для текущей фичи или всего проекта.

## Использование

```
/qa

Feature: {название фичи (опционально)}

Scope: {feature|project|release}
```

## Процесс

1. AI определяет контекст:
   - Если указана фича: читает `.specify/features/{name}/spec.md` и `tasks.md`
   - Если scope=project: проверяет все активные фичи
   - Если scope=release: проверяет фичи в текущем релизе

2. AI выполняет QA проверку:
   - Проверяет соответствие реализации спецификации
   - Ищет потенциальные баги и edge cases
   - Проверяет покрытие тестами
   - Валидирует качество кода

3. AI создает отчет:
   - Для фичи: `.specify/features/{name}/qa_report.md`
   - Для проекта: `docs/testing/qa_report_{date}.md`

## Scope варианты

### feature (по умолчанию)
Проверяет конкретную фичу:
- Соответствие spec.md
- Покрытие тестами
- Edge cases из spec.md
- Интеграция с существующим кодом

### project
Проверяет весь проект:
- Все активные фичи
- Регрессионное тестирование
- Общее качество кода

### release
Проверяет фичи в текущем релизе:
- Готовность к релизу
- Критические баги
- Performance issues

## Пример

```
/qa

Feature: Video Recording

Scope: feature
```

## Выходные артефакты

1. `qa_report.md` - отчет о QA проверке
2. Обновление `tasks.md` с найденными проблемами (если есть)
3. Рекомендации по улучшению

## Интеграция с workflow

Команда `/qa` используется:
- После `/implement` - проверка реализации
- Перед релизом - финальная проверка
- При обнаружении багов - регрессионное тестирование

## Связь с ролью QA

Команда использует инструкции из `ROLES/07_qa.md` для выполнения проверки.
