# Design System for AI Platform - Clarifications

**Статус:** Уточнения применены согласно PRD

---

## CLARIFICATIONS (ПРИМЕНЕНЫ)

### 1. Интеграция с существующим Tailwind CSS ✅

**Решение:**
- Постепенная миграция компонентов
- Сохранить обратную совместимость (старые классы на период миграции)
- Создать новые design tokens в `tailwind.config.js`
- Приоритет миграции: Кнопки → Карточки → Формы → Навигация

**Применено в spec.md:** ✅ Раздел "TECHNICAL CONSTRAINTS" обновлен

---

### 2. Состояния кнопок (disabled, loading, error) ✅

**Решение:**
- **Disabled:** Opacity 0.5, cursor: not-allowed
- **Loading:** Spinner внутри кнопки, disabled state
- **Error:** #DC2626 (красный) только для критических ошибок
- **Success:** #10B981 (зеленый) только для успешных завершений

**Применено в spec.md:** ✅ Раздел "3.4. Состояния кнопок" добавлен

---

### 3. Hover и Focus состояния ✅

**Решение (согласно WCAG 2.1 AA из PRD):**
- **Hover:** Opacity 0.9 или цвет темнее на 10%
- **Focus:** Ring 2px solid #5F6F7A (обязательно для accessibility)
- **Active:** Opacity 0.8 или цвет темнее на 15%

**Применено в spec.md:** ✅ Раздел "5.1. Разрешено" обновлен с конкретными значениями

---

### 4. Обработка ошибок и предупреждений ✅

**Решение:**
- **Error:** #DC2626 (красный) - только для критических ошибок
- **Warning:** #F59E0B (янтарный) - для предупреждений
- **Success:** #10B981 (зеленый) - для успешных действий
- Эти цвета НЕ входят в правило 60-30-10, используются только для состояний

**Применено в spec.md:** ✅ Раздел "1.4. Цвета состояний" добавлен

---

### 5. Иконки в дизайн-системе ✅

**Решение:**
- **Цвет:** #4A4A4A (текстовый цвет из 30%)
- **Размеры:** 16px (small), 20px (medium), 24px (large)
- **Правило:** Иконки НЕ используют акцентный цвет (#5F6F7A)

**Применено в spec.md:** ✅ Раздел "4.3. Иконки" добавлен

---

### 6. Формы и input элементы ✅

**Решение:**
- **Input:** Border 1px #E6E6E6, border-radius 8px, focus ring #5F6F7A
- **Label:** #1C1C1C, font-weight Medium, 14px
- **Placeholder:** #9CA3AF
- **Error state:** Border #DC2626, error message под полем
- **Success state (опционально):** Border #10B981

**Применено в spec.md:** ✅ Раздел "4.4. Формы и Input элементы" добавлен

---

### 7. Модальные окна и диалоги ✅

**Решение:**
- **Overlay:** rgba(0, 0, 0, 0.5)
- **Modal:** Белый фон, border-radius 8px, padding 24px
- **Shadow:** Только для модальных окон (0 4px 6px -1px rgba(0, 0, 0, 0.1))
- **Кнопки:** Primary справа, Secondary слева в footer

**Применено в spec.md:** ✅ Раздел "4.5. Модальные окна" добавлен

---

### 8. Навигация (меню, breadcrumbs, tabs) ✅

**Решение:**
- **Menu:** Неактивные #4A4A4A, активные #5F6F7A, hover opacity 0.8
- **Breadcrumbs:** #4A4A4A с разделителем "/"
- **Tabs:** Активные с border-bottom 2px solid #5F6F7A

**Применено в spec.md:** ✅ Раздел "4.6. Навигация" добавлен

---

### 9. Градиенты и тени ✅

**Решение:**
- **Градиенты:** Запрещены полностью (медицинский стиль = плоский дизайн)
- **Тени:** Только для модальных окон и dropdown меню
- **Карточки:** Без теней, только border 1px #E6E6E6

**Применено в spec.md:** ✅ Раздел "EDGE CASES" обновлен

---

### 10. Миграция существующих компонентов ✅

**Решение:**
- **Постепенная миграция:** Компонент за компонентом
- **Обратная совместимость:** Сохранить старые классы на период миграции
- **Приоритет:** Кнопки → Карточки → Формы → Навигация
- **Период:** 2-4 недели для полной миграции

**Применено в spec.md:** ✅ Раздел "TECHNICAL CONSTRAINTS" обновлен

---

## DECISIONS SUMMARY

1. ✅ **Tailwind Integration:** Постепенная миграция с обратной совместимостью
2. ✅ **Button States:** Disabled, Loading, Error, Success определены
3. ✅ **Hover/Focus:** Конкретные значения (opacity, ring) для WCAG AA
4. ✅ **Error/Warning Colors:** Определены цвета состояний (не входят в 60-30-10)
5. ✅ **Icons:** Цвет #4A4A4A, размеры 16/20/24px
6. ✅ **Forms:** Полный стиль для input, label, placeholder, error/success states
7. ✅ **Modals:** Стиль определен (overlay, modal, кнопки)
8. ✅ **Navigation:** Стили для menu, breadcrumbs, tabs
9. ✅ **Shadows/Gradients:** Градиенты запрещены, тени только для модальных окон
10. ✅ **Migration Strategy:** Постепенная миграция с приоритетами

---

## PRD COMPLIANCE

Все решения соответствуют PRD требованиям:
- ✅ **WCAG 2.1 AA** - Focus states обязательны (ring 2px)
- ✅ **Accessibility** - Контрастность цветов соответствует стандартам
- ✅ **Медицинский стиль** - Чистый, минималистичный дизайн
- ✅ **English only (MVP)** - Типографика поддерживает английский
