# Команда: /idea

Начинает новую фичу с создания базовой спецификации. Алиас для `/specify` с фокусом на начальную идею.

## Использование

```
/idea

Feature: {название фичи}

IDEA:
{краткое описание идеи}

GOALS:
{цели фичи}

USER VALUE:
{ценность для пользователя}
```

## Процесс

1. AI читает `.specify/constitution.md` для понимания стандартов
2. AI создает `.specify/features/{feature-name}/spec.md` с базовой структурой
3. Спецификация включает:
   - Overview (краткое описание идеи)
   - Goals (цели фичи)
   - User Value (ценность для пользователя)
   - Initial Requirements (начальные требования, которые можно расширить)
   - Success Criteria (базовые критерии успеха)

## Отличие от /specify

- `/idea` - для начальной идеи, быстрый старт, минимальная структура
- `/specify` - для полной спецификации с детальными требованиями

## Пример

```
/idea

Feature: Smart Task Prioritization

IDEA:
Помочь пользователю видеть самые важные задачи первыми через AI-приоритизацию.

GOALS:
- Автоматическая приоритизация задач
- Визуальное выделение важных задач
- Персонализация под пользователя

USER VALUE:
Пользователь тратит меньше времени на выбор следующей задачи, фокус на важном.
```

## После создания

1. Проверьте spec.md вручную
2. Используйте `/specify` для расширения спецификации детальными требованиями
3. Используйте `/clarify` для уточнения неясных моментов
4. Используйте `/plan` для создания технического плана

## Интеграция с workflow

Команда `/idea` является первым шагом в Spec-Driven workflow:
```
/idea → /specify → /clarify → /plan → /tasks → /implement
```
